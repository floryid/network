Common Vulnerabilities

Daftar kerentanan umum yang sering ditemukan dalam keamanan jaringan.

Kerentanan Umum

Pelajari jenis-jenis kerentanan umum dan bagaimana cara mendeteksinya.

1. SQL Injection (SQLi)

SQL Injection terjadi ketika perintah SQL dimasukkan ke dalam input aplikasi yang kemudian dieksekusi oleh database. Hal ini dapat mengakibatkan pencurian data atau modifikasi data.

Contoh:
Input: ' OR '1'='1
Query: SELECT * FROM users WHERE username = '' OR '1'='1';

Solusi: Gunakan prepared statements dan parameterized queries.

2. Cross-Site Scripting (XSS)

XSS terjadi ketika input yang tidak divalidasi disisipkan ke dalam halaman HTML, memungkinkan penyerang menyuntikkan skrip jahat.

Contoh:
Input: <script>alert('XSS')</script>
Efek: Popup akan muncul pada pengguna.

Solusi: Gunakan encoding pada input dan output HTML, serta validasi input dari pengguna.

3. Command Injection

Command Injection memungkinkan penyerang mengeksekusi perintah shell pada server melalui input yang tidak divalidasi.

Contoh:
Input: ; ls -la
Command: ping 127.0.0.1; ls -la

Solusi: Validasi dan sanitasi input, serta hindari penggunaan fungsi yang menjalankan perintah shell secara langsung.

4. Cross-Site Request Forgery (CSRF)

CSRF memungkinkan penyerang untuk memaksa pengguna yang terautentikasi melakukan tindakan tertentu tanpa sepengetahuannya.

Contoh:
Penyerang mengirimkan link yang menyebabkan pengguna mengirimkan permintaan POST tanpa disadari.

Solusi: Gunakan token CSRF untuk setiap permintaan yang mengubah data atau melakukan tindakan penting.

5. Directory Traversal

Directory traversal memungkinkan penyerang mengakses file yang tidak seharusnya dapat diakses dengan menggunakan karakter ".." dalam jalur direktori.

Contoh:
Input: ../../etc/passwd
Efek: Membaca file konfigurasi di server.

Solusi: Validasi jalur input dan batasi akses ke direktori tertentu.
