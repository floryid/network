WPScan

Pengenalan

WPScan adalah alat keamanan untuk memindai dan mengaudit situs WordPress. Alat ini dapat digunakan untuk mendeteksi kerentanan, plugin dan tema yang tidak aman, serta informasi tentang pengguna.

50 Contoh Penggunaan

1. Memindai kerentanan di situs WordPress

wpscan --url https://example.com

2. Menampilkan versi WordPress yang digunakan

wpscan --url https://example.com --enumerate v

3. Mencari pengguna WordPress

wpscan --url https://example.com --enumerate u

4. Mencari plugin yang terinstal

wpscan --url https://example.com --enumerate p

5. Mencari tema yang terinstal

wpscan --url https://example.com --enumerate t

6. Mencari kerentanan pada plugin tertentu

wpscan --url https://example.com --plugins-detection mixed

7. Menggunakan file daftar pengguna

wpscan --url https://example.com --usernames users.txt

8. Memindai situs dengan pengaturan proxy

wpscan --url https://example.com --proxy http://127.0.0.1:8080

9. Menyimpan hasil pemindaian ke file

wpscan --url https://example.com --output hasil.txt

10. Mencari kerentanan XSS

wpscan --url https://example.com --enumerate vp

11. Menjalankan pemindaian dengan kecepatan yang lebih lambat

wpscan --url https://example.com --delay 2

12. Memindai situs dengan autentikasi dasar

wpscan --url https://example.com --basic-auth admin:password

13. Mencari kerentanan pada tema tertentu

wpscan --url https://example.com --theme-detection aggressive

14. Menggunakan API untuk mendapatkan informasi kerentanan

wpscan --url https://example.com --api-token your_api_token

15. Menggunakan opsi untuk mendeteksi file yang dapat dieksekusi

wpscan --url https://example.com --enumerate ap

16. Mencari informasi tentang pengguna admin

wpscan --url https://example.com --enumerate u --show-counts

17. Memindai subdomain dari situs tertentu

wpscan --url https://sub.example.com

18. Mencari kerentanan di plugin tertentu

wpscan --url https://example.com --plugins-detection aggressive

19. Memindai situs tanpa menggunakan server jarak jauh

wpscan --url https://example.com --no-cache

20. Mencari direktori wp-admin

wpscan --url https://example.com --enumerate d

21. Mencari kerentanan pada plugin dan tema

wpscan --url https://example.com --enumerate pt

22. Menggunakan cookie untuk autentikasi

wpscan --url https://example.com --cookie "cookie_name=cookie_value"

23. Menjalankan pemindaian hanya untuk kerentanan yang diketahui

wpscan --url https://example.com --enumerate vp --no-update

24. Memindai direktori upload

wpscan --url https://example.com/wp-content/uploads

25. Mencari plugin yang rentan dari daftar

wpscan --url https://example.com --usernames users.txt --plugins-detection mixed

26. Mencari pengguna dengan level akses tertentu

wpscan --url https://example.com --enumerate u --role editor

27. Menampilkan hasil dengan format JSON

wpscan --url https://example.com --output hasil.json --format json

28. Menggunakan proxy untuk menyembunyikan alamat IP

wpscan --url https://example.com --proxy http://127.0.0.1:8080

29. Mencari informasi tentang patch yang tersedia

wpscan --url https://example.com --patches

30. Mencari file yang terpapar di direktori

wpscan --url https://example.com --enumerate rf

31. Mencari kerentanan di basis data

wpscan --url https://example.com --db

32. Menampilkan detail lebih lanjut tentang pengguna

wpscan --url https://example.com --enumerate u --show-counts --verbose

33. Mencari informasi tentang plugin yang tidak terpasang

wpscan --url https://example.com --plugins-detection mixed --enumerate p

34. Menggunakan daftar plugin dari file

wpscan --url https://example.com --plugins-file plugins.txt

35. Memindai situs dengan pengaturan multi-site

wpscan --url https://example.com --multi-site

36. Memindai situs untuk kerentanan SQL Injection

wpscan --url https://example.com --sql-injection

37. Mencari kerentanan yang sudah diperbaiki

wpscan --url https://example.com --enumerate vp --show-fixed

38. Memindai kerentanan pada shortcode

wpscan --url https://example.com --shortcode

39. Mengatur jumlah thread saat melakukan pemindaian

wpscan --url https://example.com --threads 10

40. Menampilkan bantuan penggunaan

wpscan --help

41. Memindai kerentanan CSRF

wpscan --url https://example.com --csrf

42. Mencari kerentanan pada tema dan plugin dengan opsi verbose

wpscan --url https://example.com --enumerate vp --verbose

43. Memindai menggunakan API dari WPScan

wpscan --url https://example.com --api-token your_api_token --enumerate p

44. Mencari tema yang tidak terupdate

wpscan --url https://example.com --enumerate t --show-outdated

45. Memindai beberapa situs secara bersamaan

wpscan --url https://example1.com,https://example2.com

46. Mencari informasi tentang pengguna admin

wpscan --url https://example.com --enumerate u --role administrator

47. Memindai kerentanan dengan laporan ke dalam file CSV

wpscan --url https://example.com --output laporan.csv --format csv

48. Memindai situs dengan opsi verbose untuk output lebih detail

wpscan --url https://example.com --verbose

49. Menentukan path dari instalasi WordPress

wpscan --url https://example.com/wp --enumerate p

50. Mencari kerentanan khusus di dalam database WPScan

wpscan --url https://example.com --db

Catatan Penting

- Selalu gunakan WPScan secara etis dan dengan izin pemilik situs web.
- Pastikan untuk memperbarui WPScan secara berkala untuk mendeteksi kerentanan terbaru.
- Gunakan hasil pemindaian untuk meningkatkan keamanan situs Anda.
